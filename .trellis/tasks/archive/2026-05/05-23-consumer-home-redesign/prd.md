# To C 首页重设计

## 目标

把当前偏 B 端工作台说明的首页，改成面向 C 端创作者的产品入口。用户进入首页后，应能快速理解“我现在可以做什么”，并被明确引导去：

- 用简单生图快速出第一张图。
- 进入画布继续做参考图、排版、编辑和 Agent 流程。
- 进入图片广场浏览公开作品，获得灵感并继续创作。

## 背景与已确认事实

- 当前首页实现位于 `apps/web/src/features/home/HomePage.tsx`，样式位于 `apps/web/src/styles/home.css`。
- 当前首页首屏强调“AI 创作工作台”“服务状态”“本地工作台”“少做切换，多留证据”“本地优先”等信息，用户观感偏工具/后台说明，不像 C 端创作产品。
- 当前 `HomePage` 只接收 `onOpenGenerate` 和 `onOpenGallery` 两个入口回调。画布和图片广场在路由中已存在，但首页未作为主入口突出。
- 当前已登录状态由 `App.tsx` 在主应用外渲染 `.auth-user-bar`，位置固定在右下角，显示“已登录：Local Admin”和退出按钮；用户要求移到右上角，只在顶部显示账户名，删除右下浮块。
- `CanvasApp` 已有路由：`home`、`generate`、`canvas`、`pool`、`gallery`、`publicGallery`、`admin`。
- 公开图片广场现有接口为 `/api/gallery/public?limit=60`，`GalleryPage` 已用该接口渲染公开作品。
- 项目视觉规范要求保留 warm paper、dark ink、copper、teal、editorial typography；首页应仍是应用入口，不做纯营销 landing。
- 用户明确要求：这是 To C 产品，用户端不应展示偏后台/服务配置的信息。

## 需求

1. 首页首屏必须转向 C 端创作语境，文案直接回答“我可以从哪里开始创作”。
2. 首屏必须有三个清晰入口：
   - 简单生成：适合一句话快速出图。
   - 画布创作：适合参考图、排版、深度编辑、Agent 流程。
   - 图片广场：适合浏览公开作品、获得灵感。
3. 首页不再展示技术状态卡、服务来源、fallback 顺序、密钥、本地运行、命令行或管理员语境。
4. 首页可以保留必要错误提示，但必须用用户能理解的创作阻塞文案，不把 Provider/API 细节当首屏卖点。
5. 首页视觉要更像 C 端创作产品：更强的灵感图像、明确 CTA、短句利益点、少解释工作流。
6. 首页应使用真实或产品内资产承载视觉。优先展示公开广场最新作品预览；没有公开作品或加载失败时回退到现有产品预览资产和静态灵感提示。
7. 移动端首屏入口必须清晰可点，不出现文字挤压、按钮换行失控或视觉遮挡。
8. 中英文 i18n 都要同步更新，不硬编码用户可见文案。
9. 登录用户信息必须进入顶部右侧导航区域，显示用户名并保留退出能力；右下角浮动登录信息块必须移除。

## 非目标

- 不改公开广场的列表页能力、排序和公开/私密机制。
- 不改简单生图、画布生成、Agent 执行、Gallery 私有管理的业务逻辑。
- 不把未登录首页/注册流程纳入本任务。
- 不新增首页直接输入提示词并跨页预填，除非后续明确追加。
- 不新增图片审核、推荐算法或埋点系统。

## 验收标准

- [ ] 首页首屏不再出现“图像服务”“OpenAI API”“Codex”“BYOK”“密钥”“fallback”“pnpm dev”等偏技术/后台文案。
- [ ] 首页首屏提供并突出“简单生成”“进入画布”“逛图片广场”三个入口。
- [ ] 点击三个入口分别能进入 `generate`、`canvas`、`publicGallery`。
- [ ] 首页有图片/作品视觉区域；有公开作品时展示公开作品预览，无公开作品时有稳定回退视觉。
- [ ] 首页文案明显面向创作者/普通用户，而不是管理员、开发者或运维用户。
- [ ] 已登录用户名称显示在顶部右侧；页面右下角不再出现登录信息浮块。
- [ ] 退出能力仍可从顶部账户区域触发。
- [ ] 桌面和移动视口下，首页首屏无文字重叠、按钮溢出或横向滚动。
- [ ] 用户可见文案同时有 `zh-CN` 和 `en`。
- [ ] `pnpm typecheck` 和 `pnpm build` 通过。
- [ ] UI 验证覆盖 `http://localhost:5173/` 的桌面和移动视口。

## 已确认决策

- 首页是否应该在首屏直接展示公开广场真实作品预览？

结论：展示，但作为灵感预览，不作为唯一视觉依赖。若接口失败或暂无公开作品，回退到产品预览图和灵感提示卡。这样更符合 C 端“先被作品吸引”的习惯，同时不让首页依赖外部数据才能成立。
