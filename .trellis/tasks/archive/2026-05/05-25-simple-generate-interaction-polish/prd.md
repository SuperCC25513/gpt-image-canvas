# 打磨简单生图交互灵敏度

## 目标

提升 `/generate` 简单生图工作台的按钮点击灵敏度和反馈清晰度，让用户感觉底部工具条、任务卡、参数按钮、生成按钮都更容易点中、更容易理解当前状态。

## 背景

当前页面视觉已经接近工作台，但用户反馈“很多按钮触发不够灵敏”。从界面结构看，可能原因包括：底部工具条横向滚动区吞掉点击意图、紧凑 chip 真实命中面积偏小、`details summary` 参数按钮反馈不够明确、生成按钮 disabled 时缺少足够解释、移动端滚动区和按钮区抢事件。

## 需求

- 底部工具条里的上传、张数、比例、风格、私密、参数、生成按钮应更容易点击。
- 紧凑按钮/图标按钮要有足够命中区，且不影响布局稳定。
- 横向滚动工具条需要减少误触，不应让用户轻点按钮时像没反应。
- 参数按钮打开/关闭要有清楚反馈。
- 生成按钮禁用时，用户应能看出原因；非禁用时点击反馈要明确。
- 左侧任务卡保持块状样式，点击命中和选中态可靠。
- 移动端底部输入器完整可见，无横向溢出，按钮不重叠。

## 不做

- 不改生成 API、积分规则、Gallery 数据结构、画布流程。
- 不引入新依赖或动画库。
- 不重做页面视觉主题。

## 验收标准

- [x] 内置 Browser 打开 `http://127.0.0.1:8787/generate`，能明确复现并解释按钮“不灵敏”的主要原因。
- [x] 上传、参数、生成、左侧任务卡的命中区和反馈改进。
- [x] 底部工具条不出现按钮重叠或横向页面溢出。
- [x] 生成按钮 disabled 状态有明确 title/视觉反馈，enabled 状态有清晰 hover/press。
- [x] 移动端 390 x 844 验证无横向溢出，底部输入器完整可见。
- [x] `pnpm typecheck` 通过。
- [x] `pnpm build` 通过。
