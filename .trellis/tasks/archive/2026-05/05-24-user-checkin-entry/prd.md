# 增加用户签到入口

## 目标

让登录用户在首页、画布、提示池、作品库等主要页面都能清楚看到积分余额，并能直接进行每日签到获取积分。

当前签到能力已经存在，但入口只出现在画布生成面板的积分卡片里。用户从首页或其他页面进入时，很难发现「签到 +N」按钮。

## 已确认事实

- 后端已有签到接口：`POST /api/checkin`。
- 后端已有账户状态接口：`GET /api/auth/me`，返回用户积分、系统积分配置和今日签到状态。
- 签到每天一次，奖励值来自后台配置「每日签到奖励」。
- 画布生成面板已有签到按钮，会调用 `/api/checkin` 并刷新本地账户状态。
- 简单生成页有积分摘要，但没有签到按钮。
- 顶部导航已经展示当前登录用户，适合作为全局可见入口。

## 需求

- 在全局顶部账号区域增加清晰的积分/签到入口。
- 登录用户应能看到当前积分余额。
- 未签到时应能点击按钮完成今日签到，并看到奖励积分数量。
- 已签到后入口应显示今日已签到，且不可重复签到。
- 签到成功后应刷新当前用户积分余额和今日签到状态。
- 签到失败时应显示可理解的错误反馈。
- 入口应在首页、画布、提示池、作品库、简单生成等主页面保持可见。
- 保留画布生成面板里的现有积分卡片，不破坏生成前积分校验。

## 验收标准

- [ ] 顶部导航或账号区域可见当前积分余额。
- [ ] 未签到用户可从顶部区域点击签到并获得每日奖励。
- [ ] 签到成功后余额增加，按钮变为今日已签到。
- [ ] 今日已签到状态下不能重复触发签到。
- [ ] `/api/checkin` 请求失败时显示错误，不静默失败。
- [ ] 首页、画布、提示池、作品库、简单生成页面均可看到入口。
- [ ] `pnpm typecheck` 和 `pnpm build` 通过。
- [ ] 浏览器验证桌面和移动视口无文字溢出或导航挤压。

## 非目标

- 不新增新的积分规则。
- 不改后端每日签到业务规则。
- 不改后台积分配置表单。
- 不新增积分流水明细页。

## 已确认方案

- 顶部账号区域直接展示「积分余额 + 签到按钮」。
- 移动端使用更紧凑的布局，避免顶部导航挤压和文字溢出。

选择直接展示的原因：用户当前问题是入口不可见，直接展示发现成本最低。
