# 积分扣费与每日签到

## Goal

新增积分余额、积分流水、生成预扣和失败退款，以及每日签到奖励。完成后，生成会按后台配置消耗积分，积分不足时给出稳定错误。

## Parent And Dependency

- 父任务：`.trellis/tasks/05-21-mysql-users-gallery`
- 前置依赖：`.trellis/tasks/05-22-user-auth-ownership` 已完成当前用户、owner 和基础积分字段。

## Requirements

- 新增系统设置：注册送积分、每张图消耗积分、每日签到奖励、单次生成数量上限。
- 新增 `credit_transactions` 表，记录用户、变动值、原因、关联生成或签到、管理员备注、创建时间。
- 新增 `user_checkins` 表，以 `user_id + checkin_date` 唯一约束保证每天只签到一次。
- 注册时按设置发放初始积分，并写积分流水。
- 生成前按 `输出数量 * 每张图消耗积分` 计算成本。
- 生成前在事务内检查余额并预扣积分。
- 积分不足返回稳定错误码和 HTTP 状态，不进入 provider 调用。
- 生成全部失败时全额退款；部分失败时按失败输出数量退款。
- 退款必须写积分流水，避免余额变化不可追踪。
- 前端显示积分余额、生成预计消耗、积分不足状态和签到入口。

## Acceptance Criteria

- [ ] 新用户注册后获得后台配置的初始积分。
- [ ] 用户每日首次签到成功增加积分；重复签到不重复加积分。
- [ ] 生成前积分不足时，API 返回稳定错误码，余额不变。
- [ ] 生成成功后余额按输出数量扣减。
- [ ] 生成全部失败后退回本次预扣积分。
- [ ] 部分输出失败后，只退回失败输出对应积分。
- [ ] 所有扣费、退款、注册赠送、签到奖励都有流水。
- [ ] Web 显示当前积分余额和签到状态。
- [ ] 通过 `pnpm typecheck`。
- [ ] 通过 `pnpm build`。
- [ ] 运行 `pnpm dev` 后，用内置浏览器验证签到、余额变化、积分不足提示。

## Out Of Scope

- 不做真实支付、充值、订单、优惠券。
- 不做复杂会员等级。
- 不做每日免费额度，除非后续单独确认。

## Notes

- 积分扣费必须用数据库事务；不能先扣内存状态再异步补流水。
- 后续后台管理任务会提供积分规则配置和管理员手动调分 UI。
