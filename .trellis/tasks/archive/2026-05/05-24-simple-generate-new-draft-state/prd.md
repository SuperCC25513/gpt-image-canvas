# 修正简单生图新建创作状态

## 目标

让 `/generate` 的左侧任务、中央结果区和“新建创作”状态一致：

- 选择左侧某个生成任务时，中央结果区显示这个任务对应的生成结果。
- 点击“新建创作”时，退出已有任务结果视图，清空中央区域，回到空白创作提示/预设入口。
- 底部输入器随“新建创作”进入空提示词状态，方便直接输入新画面描述。

## 需求

- 左侧任务卡代表最近生成结果，可点击进入对应结果。
- 当前任务卡有清晰选中态；新建创作状态下不应继续高亮旧任务。
- 中央结果区不再默认展示所有最近结果；它应展示当前选中的任务结果。
- 点击“新建创作”后，中央区域显示空状态标题、说明、预设卡和底部输入器，而不是继续显示上一张结果。
- 新生成成功后，应自动进入本次生成任务视图，并在左侧出现对应任务。
- 保持已有上传参考图、参数面板、张数/尺寸/风格/公开/积分/生成按钮能力。
- 保持现有暖色背景和上一轮布局样式。
- 移动端不出现横向溢出或底部输入器截断。

## 不做

- 不改生成 API、Gallery 数据结构、积分规则或画布流程。
- 不新增任务队列、会话存储或外部依赖。
- 不调整顶部导航和其它页面。

## 验收标准

- [x] 点击左侧任务卡后，中央区域只显示该任务对应的结果卡。
- [x] 点击“新建创作”后，中央区域回到空创作提示/预设入口。
- [x] 点击“新建创作”后，底部输入框清空，旧任务卡不再保持选中态。
- [x] 新生成成功后，中央区域显示本次生成结果，左侧任务卡同步可选。
- [x] 桌面 `/generate` 无横向溢出，左侧任务、结果区、输入器不重叠。
- [x] 移动端 `/generate` 无横向溢出，底部输入器完整可见。
- [x] `pnpm typecheck` 通过。
- [x] `pnpm build` 通过。
- [x] 使用内置 Browser 验证桌面和移动 `/generate` 通过。
