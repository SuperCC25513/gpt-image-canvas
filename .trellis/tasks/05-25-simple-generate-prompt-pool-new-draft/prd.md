# 简单生图默认新创作并复用提示池筛选

## 目标

让 `/generate` 进入页面时默认停留在新创作状态，并让新创作预设卡复用提示池的数据筛选逻辑，展示可直接套用的 GPT Image 图片提示词。

## 背景

当前 `/generate` 加载后会自动选中最近第一个作品，用户需要先点“新建创作”才看到预设。预设数据也应更贴近提示池页面，而不是混用最近作品优先逻辑。

## 需求

- 默认进入 `http://127.0.0.1:8787/generate` 时展示新创作空状态，不自动选中第一个作品。
- 左侧最近作品仍保留，点击某个作品后展示该作品对应结果。
- 点击“新建创作”回到新创作空状态，清空输入和参考图。
- 新创作预设卡复用提示池筛选逻辑：
  - 图片媒体类型。
  - 优先 `GPT Image` 模型。
  - 使用提示池同一套排序函数。
  - 不足 4 条时再从图片提示词或最近生成结果补齐。
- 预设卡只多一层“套用这个预设”入口；点击只填入输入框，不触发生成。
- 继续过滤不适合简单文生图入口的模板、参考图、上传图、过长提示词。

## 不做

- 不改提示池 API。
- 不改 Gallery 数据结构。
- 不改生成扣费、任务队列、画布流程。
- 不引入新依赖。

## 验收标准

- [x] 直接进入 `/generate` 默认显示新创作预设，不显示第一个作品结果。
- [x] 最近作品列表仍能点击并显示对应结果。
- [x] “新建创作”能从任意作品结果回到新创作空状态。
- [x] 预设卡来源优先为提示池 `GPT Image` 图片提示词，并使用提示池排序函数。
- [x] 点击预设只填充输入框，不触发生成。
- [x] 桌面和移动端无横向溢出，预设卡不重叠。
- [x] `pnpm typecheck` 通过。
- [x] `pnpm build` 通过。
- [x] 内置 Browser 验证 `http://127.0.0.1:8787/generate` 通过。

## 验证记录

- `pnpm typecheck`：通过。
- `pnpm build`：通过。
- 内置 Browser 默认进入 `/generate`：结果卡 0，预设卡 4，左侧选中 0。
- 内置 Browser 等待提示池加载后：预设卡 4，真实图片 4。
- 内置 Browser 点击最近作品：结果卡 1，左侧选中 1。
- 内置 Browser 点击“新建创作”：结果卡 0，预设卡 4，输入框清空。
- 内置 Browser 点击预设：只填入输入框，未生成结果。
- 内置 Browser 移动视口 `390 x 844`：无横向溢出，预设卡单列。
