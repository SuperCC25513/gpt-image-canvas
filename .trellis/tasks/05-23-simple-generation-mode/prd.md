# 简单生图模式

## 目标

为低门槛用户新增一个更简单的生图入口，让用户不用先理解无限画布、参考图、Agent 和复杂侧栏，也能快速输入提示词并生成图片。创作入口默认进入简单模式，同时保留高端用户可切换到现有画布模式继续使用完整工作台。

## 用户价值

- 新用户可以先完成一次文生图，再逐步理解画布能力。
- 高端用户不丢失现有画布、参考图编辑、Agent、历史、Gallery、Provider 配置等能力。
- 简单模式复用现有生成契约和本地资产链路，避免形成第二套生成规则。

## 已确认事实

- 现有画布手动面板支持文生图和参考图编辑两种模式。
- 现有生成请求字段包括：`prompt`、`presetId`、`sizePresetId`、`size`、`quality`、`outputFormat`、`count`、`isPublic`。
- 当前默认尺寸为 `portrait-4k`，默认质量为 `high`，默认输出格式为 `png`，默认数量为 `1`。
- 尺寸预设来自 `packages/shared/src/image.ts`，包括 1K、2K、4K 的方图、竖版、横版、视频封面等场景。
- 风格预设包括：无风格、写实、产品、插画、海报、头像。
- 数量支持 `1`、`2`、`4`、`8`、`16`，其中 `8` 和 `16` 现在放在更多数量中。
- 生成前需检查登录/账号、积分、最大张数、提示词、尺寸有效性、提供方可用性。
- 成功生成后已有历史记录、画布占位替换、本地资产和 Gallery 链路。

## 需求

- 新增“简单模式”和“画布模式”的切换入口。
- 保留当前首页，不用简单模式直接替换 `/` 首页。
- 新增独立简单生成路由 `/generate`。
- 从创作入口进入时默认使用简单模式，即进入 `/generate`。
- 顶部导航和首页创作入口应能进入简单生成，`/canvas` 继续作为完整画布模式。
- 简单模式应围绕低门槛文生图主流程组织，而不是要求用户先理解 tldraw 画布。
- 第一版简单模式只覆盖文生图；参考图编辑继续留在画布模式。
- 简单模式参数参考现有画布手动面板，但默认只展示低认知负担字段：
  - 提示词。
  - 场景/尺寸预设。
  - 生成数量。
  - 风格预设。
  - 生成按钮。
- 简单模式默认参数采用低门槛试错配置：`square-1k`、`auto` 质量、`png` 输出格式、`1` 张、无风格。
- 画布模式保留现有默认参数，不因简单模式默认值改变。
- 简单模式可把质量、输出格式、公开开关、自定义尺寸放入高级区域，默认折叠或弱化。
- 简单模式必须复用现有生成接口、校验、错误提示、积分/账号限制和 Provider 状态展示。
- 简单模式生成结果需要停留展示在简单模式结果区，可下载、可复用提示词，并保持与 Gallery / 本地资产记录关联。
- 简单模式结果区展示最近生成的 8 张图片，桌面布局优先采用 2x4。
- 简单模式结果区提供“查看更多”入口，点击后进入作品/Gallery 路由 `/gallery`；开发环境对应 `http://127.0.0.1:5173/gallery`。
- 简单模式生成完成后可以提示用户去画布继续编辑，但不能自动跳转到画布区。
- 画布模式保留现有完整能力，包括参考图编辑、Agent、生成历史、画布定位、Prompt Pool / 收藏提示词等。
- 用户从简单模式主动切到画布模式后，应能继续查看或使用本次生成结果，避免结果只停留在简单页面。
- 用户在简单模式结果区点击“去画布继续编辑”后，应进入画布并自动插入本次生成结果。
- 重复点击“去画布继续编辑”不应无提示地重复插入同一批结果。
- 用户可随时从画布模式返回简单模式。
- 新增用户可见文案需同步 `zh-CN` 和 `en`。

## 验收标准

- [ ] 从创作入口进入时默认进入简单模式，而不是直接进入完整画布操作界面。
- [ ] 当前首页 `/` 保留，不被简单模式替换。
- [ ] 简单模式有独立路由 `/generate`，创作入口默认进入该路由。
- [ ] 页面上存在清晰的简单/画布模式切换，当前模式状态可辨认。
- [ ] 简单模式可输入提示词并使用现有生成服务成功生成图片。
- [ ] 简单模式至少支持选择尺寸预设、风格预设、生成数量。
- [ ] 简单模式初始默认值为 `square-1k`、`auto` 质量、`png`、`1` 张、无风格。
- [ ] 简单模式的高级参数可设置质量、输出格式、公开开关或等价现有字段。
- [ ] 缺少提示词、账号不可用、积分不足、提供方不可用、尺寸无效等错误能在简单模式清晰展示。
- [ ] 简单模式生成结果停留在简单模式结果区，并提供明确提示或动作让用户主动进入画布继续编辑。
- [ ] 简单模式结果区展示最近生成的 8 张图片，桌面为 2x4 或等价稳定网格，并可通过“查看更多”进入 Gallery 作品页。
- [ ] 简单模式生成结果进入现有本地资产/Gallery/历史链路，且用户主动切换画布模式后可继续使用。
- [ ] 用户主动从简单模式结果进入画布时，本次结果会插入画布或定位到已插入结果，不产生不可控重复图片。
- [ ] 切换到画布模式不破坏现有画布、Agent、参考图编辑和历史功能。
- [ ] 移动端和桌面端简单模式无文本溢出、控件遮挡或不可点击问题。
- [ ] `pnpm typecheck` 和 `pnpm build` 通过。
- [ ] UI 验证覆盖 Vite 应用桌面视口和移动视口。

## 暂不包含

- 简单模式第一版不做参考图上传/选图编辑。
- 不重做 Agent 规划体验。
- 不新增图片模型或 Provider 配置规则。
- 不改变现有生成接口字段语义。
- 不删除画布模式中的高级能力。

## 已确认决策

- 简单模式生成后的结果主归宿是简单模式结果区；可以提示用户进入画布，但不能自动跳转。
- 简单模式 MVP 先只做文生图；参考图编辑留在画布模式。
- 不直接替换当前首页。
- 新增 `/generate` 作为简单模式独立路由；`/canvas` 保持完整画布。
- 简单模式结果区的“去画布继续编辑”是用户主动动作；点击后进入画布并插入或定位本次结果。
- 简单模式结果区展示最近生成的 8 张图片，桌面优先 2x4；“查看更多”进入 `/gallery` 作品页。
- 简单模式默认参数为 `square-1k`、`auto`、`png`、`1` 张、无风格；画布模式继续保留自己的默认值。
