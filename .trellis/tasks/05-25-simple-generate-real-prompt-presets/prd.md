# 替换简单生图预设假图

## 目标

把 `/generate` 空状态中的 4 个预设卡从低质 CSS 假图改为真实提示词入口。用户应看到真实、有参考价值的提示词和作品图，而不是抽象占位图。

## 背景

当前 4 个预设卡封面是 CSS 画出来的抽象图，质感差，和生图产品预期不匹配。用户建议“还不如直接使用提示词中的随机 4 个”。

## 需求

- 空状态预设卡优先从已有生成记录/提示词来源中取 4 个真实提示词。
- 有对应生成图片时，预设卡封面使用真实作品缩略图。
- 没有真实图片时，不再使用 CSS 假插画；使用干净的文字型提示词卡。
- 点击预设卡仍填充底部输入框，不触发生成。
- 卡片需要保留比例/张数等轻量元信息，但信息来源应来自真实记录或提示词。
- 刷新页面时可随机变化，但单次渲染内保持稳定，不因输入造成反复跳动。
- 保持现有暖色背景、居中工作台和底部输入器布局。

## 不做

- 不调用生成接口，不消耗额度。
- 不改 Gallery/API 数据结构。
- 不引入外部图片源或新依赖。

## 验收标准

- [x] `/generate` 空状态不再显示当前 CSS 假预览图。
- [x] 有图库数据时，预设卡展示随机 4 个真实提示词/作品图。
- [x] 无图库数据时，预设卡仍可用，并展示文字型提示词卡。
- [x] 点击任意预设卡只填充输入框，页面不跳动、不生成图片。
- [x] 桌面和移动端无横向溢出，卡片内容不重叠。
- [x] `pnpm typecheck` 通过。
- [x] `pnpm build` 通过。
- [x] 内置 Browser 验证 `http://127.0.0.1:8787/generate` 通过。

## 验证记录

- `pnpm typecheck`：通过。
- `pnpm build`：通过。
- 内置 Browser 桌面视口：4 张预设卡，真实图片 4 张，假图节点 0，横向溢出 false。
- 内置 Browser 点击预设：只填入输入框，结果卡数量保持 0。
- 内置 Browser 移动视口 `390 x 844`：4 张预设卡单列排列，假图节点 0，横向溢出 false。
