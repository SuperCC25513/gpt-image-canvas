# 兑换码积分兑换

## Goal

新增积分兑换码能力。管理员可以在后台创建、批量创建、查看和管理兑换码，用户可以在积分中心输入兑换码兑换积分。兑换成功后，用户余额增加，并形成可追踪的积分流水。

用户价值：

- 用户获得除注册赠送、每日签到之外的可控积分获取方式。
- 管理员可以运营发放积分，不需要手动逐个调整用户余额。
- 积分变更继续可审计，避免余额变化不可追踪。

## Confirmed Facts

- 当前积分余额存于 `users.credits`。
- 当前积分流水表是 `credit_transactions`，已有类型：`registration_bonus`、`daily_checkin`、`generation_charge`、`generation_refund`、`admin_adjustment`。
- 当前用户侧积分明细接口为 `GET /api/credits/transactions?limit=30`，积分中心 `/credits` 展示余额、每日签到和最近流水。
- 后台已有用户积分调整和系统积分配置能力。
- 项目同时支持 SQLite 和 MySQL；新增持久化表需要同步两种初始化逻辑。
- 可靠性约束：任何积分余额变更必须在数据库事务中同时写入 `credit_transactions`。
- 安全约束：`/api/admin/*` 必须要求管理员会话；兑换码输入需要服务端校验，不能在前端信任。

## In Scope

- 新增兑换码数据模型，支持 SQLite 和 MySQL。
- 新增用户兑换接口，登录用户在积分中心提交兑换码。
- 兑换成功后增加用户积分，并写积分流水。
- 防止同一用户重复兑换不允许重复使用的兑换码。
- 兑换码采用一次性全局码：一个码只能被一个用户成功兑换一次；批量发放给多个用户时由后台批量生成多个码。
- 后台管理兑换码列表。
- 后台兑换码管理放在 Admin 独立「兑换码」Tab，路由为 `/admin/redemption-codes`，导航顺序放在「用户」之后。
- 后台创建单个兑换码。
- 后台批量创建兑换码。
- 后台可以删除未兑换且无积分流水关联的兑换码；已兑换或已产生流水关联的码不能硬删除。
- 后台可以禁用/启用兑换码；禁用后不能兑换，但保留列表和审计线索。
- 后台创建兑换码时可以设置兑换成功后增加的积分额度。
- 单个兑换码积分额度必须是 1 到 10000 的整数。
- 后台同一入口支持生成 1 个或批量生成多个兑换码。
- 后台批量创建一次最多生成 200 个兑换码。
- 后台设置兑换码过期时间时提供快捷选项：明天、3 天、一个星期、一个月、一年；快捷选项写入的时间固定为对应日期的 23:59:59。
- 后台生成数量为 1 时，创建成功后自动复制该兑换码。
- 第一版不允许管理员手动输入自定义码值；单个和批量创建都由系统自动生成码值。
- 系统生成码格式为 `CC-XXXX-XXXX-XXXX`，只使用大写字母和数字，并排除容易混淆的 `O/0/I/1`。
- 后台兑换码列表提供复制按钮，方便复制码值。
- 前端积分中心新增兑换码输入入口，并在兑换成功后刷新余额和流水。

## Requirements

- 兑换码必须有码值、积分额度、状态、创建时间。
- 兑换码支持可选过期时间；不填表示永久有效，填写后到期不可兑换。
- 兑换码兑换失败时必须返回稳定错误，用户能看到可理解提示。
- 兑换码相关 API 必须复用现有认证边界：用户兑换需要登录，后台管理需要管理员。
- 兑换码列表至少显示码值、积分额度、状态、已兑换情况和创建时间。
- 批量创建后应展示新创建的码列表，并允许复制。
- 兑换产生的积分明细需要有清晰文案。
- 用户侧积分流水只显示兑换码短码；后台兑换码列表显示完整码值。

- [ ] 管理员可以删除未兑换且无积分流水关联的兑换码。
- [ ] 已兑换或已产生流水关联的兑换码不能被删除，并显示明确错误。
- [ ] 管理员可以禁用/启用兑换码。
- [ ] 已禁用兑换码不能兑换，并显示明确错误。
- [ ] 用户可以在积分中心输入兑换码兑换积分。

## Out Of Scope

- 不做支付、充值或兑换码购买。
- 不做公开网络级风控、短信验证或验证码。
- 不改变现有注册、签到、生成扣费、失败退款规则。
- 不导出 CSV，除非后续明确要求。

## Decisions

- 兑换码使用规则：一次性全局码。一个码只能被一个用户成功兑换一次。
- 过期时间：支持可选过期时间；不填永久有效，填写后到期不可兑换。
- 删除规则：只允许删除未兑换且无积分流水关联的兑换码；已兑换码保留用于审计。
- 禁用规则：后台支持禁用/启用；禁用后不可兑换。
- 生成规则：后台可生成 1 个或批量生成多个兑换码；创建时设置积分额度。
- 批量上限：一次最多生成 200 个兑换码。
- 过期快捷选项：后台创建表单提供明天、3 天、一个星期、一个月、一年，快捷选项统一设置到目标日期 23:59:59。
- 复制规则：生成数量为 1 时自动复制新码；列表仍提供复制按钮。
- 码值规则：第一版不支持手动输入自定义码；系统生成 `CC-XXXX-XXXX-XXXX` 格式码，排除易混淆字符。
- 积分额度：单码积分为 1 到 10000 的整数。
- 流水展示：用户侧流水显示短码，后台兑换码列表显示完整码。
- 后台入口：新增 Admin 独立「兑换码」Tab，路由 `/admin/redemption-codes`，放在「用户」之后。
